---
title: "Theming"
description: "How Mais Hábito implements dark and light themes using Styled Components."
---

Mais Hábito supports dark and light modes through Styled Components' `ThemeProvider`. Theme tokens are defined in `src/styles/theme.ts`, the active theme is controlled by `useThemeStore`, and TypeScript augmentation in `src/types/styled.d.ts` gives every styled component full type safety on theme properties.

## How the theme is applied

`main.tsx` reads `isDark` from `useThemeStore` and selects either `darkTheme` or `lightTheme`. The chosen object is passed to `ThemeProvider`, which makes it available to every styled component in the tree via the `theme` prop.

```typescript
const Root = () => {
  const isDark = useThemeStore((s) => s.isDark);
  const theme = isDark ? darkTheme : lightTheme;

  return (
    <ThemeProvider theme={theme}>
      <GlobalStyle />
      <App />
    </ThemeProvider>
  );
};
```

Because `useThemeStore` is a Zustand store, calling `toggleTheme()` triggers a re-render of `Root`, swapping the theme object passed to `ThemeProvider`. All styled components beneath it update immediately.

## TypeScript augmentation

`src/types/styled.d.ts` merges the `Theme` type (derived from `darkTheme`) into Styled Components' `DefaultTheme` interface:

```typescript
import 'styled-components';
import type { Theme } from '../styles/theme';

declare module 'styled-components' {
  export interface DefaultTheme extends Theme {}
}
```

This means the `theme` object inside any styled component is fully typed — your editor will autocomplete token names and report type errors if you reference a property that doesn't exist.

## Using theme tokens in styled components

Access token values via the `theme` prop interpolation:

```typescript
import styled from 'styled-components';

const Card = styled.div`
  background: ${({ theme }) => theme.colors.surface};
  color: ${({ theme }) => theme.colors.textPrimary};
  border: 1px solid ${({ theme }) => theme.colors.border};
  border-radius: ${({ theme }) => theme.borderRadius.md};
  padding: ${({ theme }) => theme.spacing.lg};
`;
```

<Tip>
  Always use theme tokens instead of hardcoded hex values. Components that reference `theme.colors.*` automatically adapt when the user switches between dark and light mode — no conditional logic required.
</Tip>

## Toggling the theme

The `useThemeStore` hook exposes `isDark` and `toggleTheme`. The `Sidebar` (desktop) and `BottomNav` (mobile) both render a toggle button using this hook:

```typescript
import { useThemeStore } from '../../store/themeStore';

const { isDark, toggleTheme } = useThemeStore();

// In JSX
<button onClick={toggleTheme}>
  {isDark ? <Sun size={18} /> : <Moon size={18} />}
</button>
```

The selected theme is persisted to `localStorage` under the key `theme` (`"dark"` or `"light"`), so the preference is restored on the next visit. If no value is stored, the app defaults to dark mode.

## Global styles

`GlobalStyle.ts` uses `createGlobalStyle` from Styled Components to apply base resets that consume theme tokens:

```typescript
export const GlobalStyle = createGlobalStyle`
  *, *::before, *::after {
    box-sizing: border-box;
    margin: 0;
    padding: 0;
  }

  html, body {
    height: 100%;
  }

  body {
    font-family: ${({ theme }) => theme.fonts.family};
    background-color: ${({ theme }) => theme.colors.background};
    color: ${({ theme }) => theme.colors.textPrimary};
    -webkit-font-smoothing: antialiased;
  }

  a {
    text-decoration: none;
    color: inherit;
  }

  button {
    cursor: pointer;
    border: none;
    background: none;
    font-family: inherit;
  }

  input, textarea {
    font-family: inherit;
  }
`;
```

`GlobalStyle` is rendered inside `ThemeProvider` in `main.tsx`, so it has access to the same active theme object as every other styled component.
